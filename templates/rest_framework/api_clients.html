API clients handles the underlying details of how network requests are made and how responses are decoded. We are using Core API which is a format-independent Document Object Model for representing Web APIs. It's recommended to use below Core API clients for more robust and meaningful way to interact with your JASPAR API rather than constructing HTTP GET requests and decoding responses.

We are providing three clients:

  • Command line client
  • Python client
  • JavaScript client

Command line client

# Install the command line client
$ pip install coreapi-cli

# Load the schema document
$ coreapi get http://127.0.0.1:8000/api/v1/matrix/

# Interact with the API endpoint
$ coreapi action v1 list

Python client

# Install the Python client library
$ pip install coreapi
import coreapi

# Initialize a client & load the schema document
client = coreapi.Client()
schema = client.get("http://127.0.0.1:8000/api/v1/doc/")

# Interact with the API endpoint
action = ["v1", "list"]
result = client.action(schema, action)

JavaScript client

<!-- Load the JavaScript client library -->
<script src="/static/rest_framework/js/coreapi-0.1.0.js"></script>
<script src="/api/v1/doc/schema.js"></script>
Interact with API. 

var coreapi = window.coreapi  // Loaded by `coreapi.js`
var schema = window.schema    // Loaded by `schema.js`

// Initialize a client
var client = new coreapi.Client()

// Interact with the API endpoint
var action = ["v1", "list"]
client.action(schema, action).then(function(result) {
    // Return value is in 'result'
})
